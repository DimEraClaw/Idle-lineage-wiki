# 資料結構說明

## 目前已存在的資料集合

### 1. EQUIP_DATA
位於 [wiki.html](wiki.html) 內，主要用於裝備百科頁。

內容包含：
- 裝備 ID
- 名稱
- 類型與子類型
- 裝備槽位
- 價格、重量、安全性
- 使用職業需求
- 描述
- 稀有度
- 屬性數值
- 掉落與取得來源
- 分類與群組資訊

用途：
- 渲染裝備卡片
- 支援搜尋與篩選
- 支援詳細資料 modal

### 2. CRAFT_DATA
位於 [wiki.html](wiki.html) 內，主要用於製作百科頁。

內容包含：
- 製作配方
- NPC 資訊
- 物品資料庫
- 材料掉落資料庫

用途：
- 顯示製作配方清單
- 顯示材料需求與來源
- 支援配方搜尋與展開顯示

### 3. REGIONS_DATA
位於 [wiki.html](wiki.html) 內，主要用於怪物與卡片百科。

內容包含：
- 地區鍵值與名稱
- 地區屬性統計
- 怪物列表
- 怪物掉落物
- 地圖位置資訊

用途：
- 顯示地區與怪物資料
- 支援關鍵字搜尋
- 提供卡片與怪物來源查詢

## 資料管理建議
- 目前資料以靜態陣列與物件形式直接存在於 [wiki.html](wiki.html) 中。
- 未來若要擴充，可考慮把資料拆成獨立 JSON 或 JavaScript 模組，降低維護成本。
- 任何新增資料都應保持結構一致，避免混用不同格式。
