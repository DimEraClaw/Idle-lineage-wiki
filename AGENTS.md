# AGENTS

## 專案目的
這個專案是一個靜態網站專案，主要用來整理與展示遊戲資料，包含遊戲主介面與百科內容。未來任何修改都應以維持現有功能與可讀性為優先。

## 可以修改的檔案
- [README.md](README.md)
- [AGENTS.md](AGENTS.md)
- [docs/](docs/)
- 需要時可修改根目錄下的 Python 資料整理腳本

## 不要修改的檔案
以下檔案應視為現有網站功能主體，除非明確要求並完成分析，否則不要直接修改：
- [index.html](index.html)
- [wiki.html](wiki.html)
- [css/](css/)
- [js/](js/)
- [afk-*.js](afk-analytics.js)
- 任何現有 HTML、CSS、JavaScript 的互動行為與樣式

## 修改前必須先分析
在任何功能修改前，必須先：
1. 讀取目前專案結構與入口檔案。
2. 確認相關資料來源與依賴。
3. 確認是否會影響現有網站功能。
4. 先規劃，再修改。

## 修改原則
- 不要一次大規模重構。
- 不要直接重寫現有 HTML、CSS、JavaScript。
- 不要自行虛構遊戲資料。
- 所有新增內容都應建立在文件或資料整理流程中，而不是直接破壞現有功能。

## 測試要求
每完成一個功能，都必須進行基本測試：
- 先確認頁面可正常開啟。
- 所有涉及的 Button 必須實際點擊測試。
- Console 必須沒有 Error。
- Network 不可有 404。

## 開發流程
1. 先分析目標與影響範圍。
2. 建立小步驟計畫。
3. 實作最小改動。
4. 重新測試。
5. 確認沒有造成回歸問題後再進一步擴充。
