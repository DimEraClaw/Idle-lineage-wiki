# 開發規則

## Coding Style
- 優先維持現有程式碼風格與結構。
- 新增內容時，盡量保持簡潔、可讀與可維護。
- 不要為了美觀而重寫整個檔案。
- 保留現有 HTML、CSS、JavaScript 的功能行為，避免無意義回歸。

## Git
- 修改前先確認工作區狀態。
- 不要在未分析情況下直接提交或切換分支。
- 新功能建議在專用分支上進行。
- 不要在未完成驗證前提交。

## 測試流程
每次完成修改後，至少檢查：
1. 頁面可正常開啟。
2. 相關功能可正常操作。
3. 按鈕與連結可正常點擊。
4. Console 無 Error。
5. Network 無 404。

## 修改流程
1. 先分析相關檔案與資料來源。
2. 先規劃要改動的範圍與影響面。
3. 優先進行最小改動。
4. 驗證結果後再考慮擴展。

## 禁止事項
- 不要直接修改現有 HTML、CSS、JavaScript 的網站功能。
- 不要一次大規模重構。
- 不要自行虛構遊戲資料。
- 不要在沒有驗證的情況下聲稱功能已完成。
- 不要忽略 Console 與 Network 錯誤。
